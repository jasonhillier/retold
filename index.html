Application building blocks for the story obsessed.

Data contains so many stories. These libraries are born from an unhealthy love of seeking out stories. Stories should be told over and over... boilerplate code should not so much. Retell what you want.

Every truth has two sides; it is as well to look at both, before we commit ourselves to either.

Fable — Core Services

[ github ]

It is tiring to setup logging and settings management in every application you write. Fable provides a single line solution to have simple logging to the console via bunyan. Add a simple configuration object and it can also write the log to a file. Or even mongodb!

Orator — Restful Web Server

[ github ]

Science has proven time and again .. restify is pretty awesome. But using it in each application server or microservice turned into the same 25 or 50 lines of boilerplate code. With orator you can just configure what you want.

Meadow — Data Access & Automatic Endpoints

[ documentation ] [ github ]

Who doesn't love writing the same code over and over again? Good question. Anybody who doesn't probably wants something to do simple data access stuff. And some of the complicated interactions as well. Meadow aims to provide a simple &lquot;magic where you want it, programmability where you don't&rquot; pattern.

Foxhound — Programmatic Query Generation

[ github ]

Write the same code. Generate queries for mongo, mysql or whatever.

Stricture — Markdown Inspired Data Definition Layer

[ github ]

Scribble a few words down and voila! Database create statements, documentation, graphs, mock objects for unit tests and a bunch more. Plays well with meadow.
